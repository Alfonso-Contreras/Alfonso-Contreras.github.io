Épicas

Registro de clientes y mascotas: El sistema debe permitir registrar a los clientes y sus mascotas en la base de datos del sistema web, incluyendo información como nombre, dirección, número de teléfono y correo del cliente, y nombre, especie, raza y edad de la mascota.

Registro de historiales médicos: El sistema debe registrar los historiales médicos de las mascotas en la base de datos del sistema web, incluyendo información como síntomas, el diagnóstico, el tratamiento y las notas del veterinario.

Registro de citas y programaciones: El sistema debe incluir la creación de un calendario en línea para poder programar citas para las mascotas.

Gestión de datos: El sistema debe incluir la creación de una interfaz de gestión de datos para que se pueda ver y editar los registros en la base de datos del sistema web, además de permitir la realización de copias de seguridad.

Generación de reportes: El sistema debe incluir la creación de una funcionalidad para generar informes y estadísticas sobre la base de datos del sistema web.

Declaración de la visión del proyecto

Desarrollar un sistema web que ofrezca una solución integral y fácil de usar que permita a los veterinarios y al personal de la clínica acceder a los datos de los pacientes, llevar un seguimiento de las citas y tratamientos, y gestionar toda esta información de forma efectiv para que, de esta manera que se pueda ofrecer una atención al cliente de alta calidad y mejorar la calidad de los servicios que se brindan a los pacientes. Además, se espera que el sistema contribuya a aumentar la rentabilidad y eficiencia de la veterinaria, ayudando a la clínica a crecer y expandirse en el mercado de la salud animal.

Métodos de priorización de historias del usuario

El método utilizado para calcular la prioridad de cada historia de usuario será el Esquema de priorización MoSCoW—El esquema de priorización MoSCoW obtiene su nombre de la versión en inglés de las frases: “Debe tener” (Must have), “Debería tener” (Should have), “Podría tener” (Could have) y “No tendrá” (Won’t have). Las etiquetas están en orden de prioridad decreciente con historias de usuario con características de “debería tener”, siendo aquellas sin las que el producto no tendrá valor, e historias de usuarios con características de “gustaría que tuviera” siendo aquellas que, a pesar de que sería bueno tener, no se es necesario incluir.
